# Query Planner

The query planner (optimizer) decides *how* to execute a SQL statement: which
[[Indexing]] structures to use, the join order for [[SQL Joins]], and whether to
scan or seek.

This note deliberately has **no YAML frontmatter** — the parser should fall back to
the filename ("Query Planner") for the title and to the top-level folder
("Databases") for the subject.
